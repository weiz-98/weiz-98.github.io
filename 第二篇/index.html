models/

  • const catSchema = new Schema(...): 定義了 Cat 的 Mongoose 模型結構，設定了 name 和 weight 兩個字段都為必須的，並啟用了自動時間戳記錄（timestamps: true），這會自動添加 createdAt 和 updatedAt 這兩個字段。
  • export default mongoose.models.Cat || model('Cat', catSchema): 導出 Cat 模型。這條語句首先檢查 mongoose.models 中是否已經存在 Cat 模型（避免重複定義），如果不存在，則使用 model('Cat', catSchema) 創建新的模型。
  • 在 mongoose.models.Cat 中，Cat 指的是 Mongoose 的模型。當你在 Mongoose 中定義一個模型時，例如通過 mongoose.model('Cat', catSchema)，Mongoose 會在其內部 models 對象上註冊該模型。所以 mongoose.models.Cat 是在查找由 mongoose.model 創建並註冊到 Mongoose 的模型。
  • 在 import { Cat } from '../types/cat' 中，Cat 指的是一個 TypeScript 類型，它在 ../types/cat 文件中定義。這個類型用於在 TypeScript 中進行靜態類型檢查，並不會在運行時存在。

這兩者是相關聯的，但用於不同的目的：

  • Mongoose 的 Cat 模型用於與 MongoDB 交互，並且會包含像是查詢、保存、更新文檔等方法。
  • TypeScript 的 Cat 類型用於在編譯時期檢查代碼，確保你正確地使用與貓咪相關的資料結構。

  • export const establishConnection = (connectionString: string) => { ... }: 導出一個函數，它接受一個 MongoDB 連接字符串，並使用該字符串來建立資料庫連接。

repo/cat.ts

  • 定義資料庫操作函數：
    • getCats: 定義一個函數，當被調用時，它會從資料庫中找到所有的貓咪資料並返回一個承諾（Promise）。
    • addCat: 接受一個貓咪物件作為參數，該物件是 Cat 型別的部分集合（使用了 Partial），並創建一個新的貓咪記錄在資料庫中。
    • getCatById: 根據給定的 ObjectId 從資料庫中尋找一個貓咪的資料。
    • updateCatById: 接受一個 ID 和一個更新物件，該物件也是 Cat 型別的部分集合，並更新相應的貓咪記錄。
    • deleteCatById: 根據提供的 ID 從資料庫中刪除一個貓咪的記錄。
  • await CatModel.findOneAndUpdate({ _id: id }, update, { new: true }): 異步等待 Mongoose 的 findOneAndUpdate 方法的執行。這個方法尋找一個匹配 _id 的文檔，用 update 物件進行更新。
  • new: true 是一個選項，當設置為 true 時，findOneAndUpdate 方法將會返回更新後的文檔。如果不設置或者設為 false，它會返回原始文檔。
  • 當 findOneAndUpdate 方法在資料庫中匹配不到任何文檔時，它將返回 null。由於這個方法本身返回一個 Promise，因此當沒有找到匹配的文檔時，你實際上得到的是一個解析為 null 的 Promise。
  • Types.ObjectId 是 Mongoose 中的一種數據類型，對應於 MongoDB 中的 ObjectId。每個在 MongoDB 中儲存的文件都有一個唯一的 _id 屬性，其值為 ObjectId。這個值是自動生成的，並且通常用於檢索和操作特定的文檔。
